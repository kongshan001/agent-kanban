# v0.5 迭代记录

## 日期: 2026-05-02

## 本轮核心: 评估优化 + 一键推进

### ✅ 已完成

#### 1. 综合评估模式 (`evaluate_single`)
- 合并 4 角色评估为 **单次 LLM 调用**
- 从综合报告中提取各维度分数
- 使用: `kanban task evaluate TK-001 --single`
- 效率提升: 4 次 LLM 调用 → 1 次，耗时从 ~2min 降到 ~30s

#### 2. Hermes 输出解析优化
- `_clean_hermes_output` 重写：只保留第二个 "## 评估报告" 之后的内容
- 过滤 TUI 装饰符、prompt 模板残留
- 遇到 "Resume this session" 标记停止

#### 3. `kanban task auto` 一键推进
- 从当前状态自动推进到需要用户介入的节点
- 自动创建 worktree → 检查规划 → 检查执行 → 运行评估
- 缺少文档时自动停住，提示用户补充
- 评估不通过自动进入自迭代

#### 4. LLM 调用统一化
- `_call_llm()` 统一入口: claude → hermes → opencode 依次 fallback
- evaluator 和 runner 共用同一调用逻辑

### 验证结果
- `kanban task auto TK-001` 自动走完 DRAFT → PLANNING → EXECUTING → EVALUATING
- 缺少文档时正确停住并提示
- `--single` 模式评估效率大幅提升

### 发现的问题
1. auto 评估阶段可能耗时较长（hermes ~30s），CLI 体验需要异步化
2. auto 命令中没有自动调用 agent 来补充缺失文档（只是停住提示）
