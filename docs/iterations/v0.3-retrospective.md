## 发现的新问题

### P1 - 需要修复

#### 1. LLM 评估在 claude CLI 不可用/超时时退化为占位
- **现象**: 如果 claude 进程启动慢或不可用，评估分数全是 1.0，必然触发自迭代
- **v0.4 方案**: 支持配置评估模式：`llm`（调用 CLI）、`manual`（人工填写）、`skip`（跳过评估）

#### 2. approve 后 worktree 未自动清理
- **现象**: approve 只做了 merge + archive，没清理 worktree
- **v0.4 方案**: approve 加 `--cleanup` 默认清理

#### 3. 迭代轮次在模拟场景中不准确
- **现象**: TK-003 显示 iteration=2 但实际只跑了一轮
- **原因**: evaluate 命令在未通过时会自增 iteration
- **v0.4 方案**: iteration 只在 SELF_IMPROVE → EXECUTING 时才增加

### P2 - 架构改进

#### 4. 评估角色太多，上下文重叠
- 按"Sub-Agent vs Agent Team"文章的建议，4 角色评估是按角色拆而非上下文边界
- **v0.4 方案**: 合并为单 Agent 综合评估，输出一份统一报告

#### 5. 缺少 `kanban task auto` 一键推进命令
- 用户需要手动跑 plan-done → exec-done → evaluate → approve
- **v0.4 方案**: `kanban task auto TK-001` 自动从当前阶段推进到需要用户介入的节点
